# rustgrid-agent v1.3.0

This feature release prioritizes complete, validated mission delivery while
adding detailed live execution telemetry and hardening worker lifecycle,
authentication, and recovery behavior.

## Install or upgrade

On macOS with Homebrew:

```sh
brew update
brew upgrade rustgrid-agent
```

For a new installation:

```sh
brew install RustGrid/tap/rustgrid-agent
```

## Mission execution

- Classify coding missions after repository checkout so the worker can use the
  actual source tree and applicable repository instructions.
- Keep multidimensional execution budgets advisory. Budget thresholds remain
  visible in telemetry but no longer interrupt Codex, discard active context,
  or terminate a mission before a complete validated change.
- Separate Codex-owned discovery, implementation, and focused validation from
  worker-owned dependency bootstrap, full quality gates, publication, and CI.
- Reuse valid dependency and gate results, and provide compact failure context
  for repair attempts without replaying the complete tool history.
- Require explicit implementation readiness and validation evidence tied to the
  current source-tree hash before successful delivery.

## Reliability and authentication

- Prevent duplicate active attempts for one ticket and prevent multiple local
  execution processes from owning the same worker identity.
- Report sandbox preparation immediately and bound sandbox commands so ticket
  assignment cannot appear frozen indefinitely.
- Retry only explicit in-flight idempotency conflicts and preserve exact
  request-body identity across lifecycle updates.
- Remove worker ID and API-key environment authentication. Run
  `rustgrid-agent login` before upgrading managed workers; identity comes from
  the generated configuration and credentials come from the OS keychain or
  owner-only credential store.

## Telemetry

- Connect detailed Codex telemetry to the live streaming execution path.
- Distinguish agent sessions, provider inference turns, model and tool calls,
  peak context, cached and uncached input, output, reasoning, duration, and
  legacy aggregate consumption.
- Preserve complete raw worker-gate evidence locally while sending bounded,
  normalized failure summaries into repair context.

## Deployment status

Artifact publication is not production deployment approval. Complete staging
certification against the exact `v1.3.0` image digest before deploying it.

See [known limitations](https://github.com/RustGrid/rustgrid-agent/blob/v1.3.0/docs/known-limitations.md)
and [production operations](https://github.com/RustGrid/rustgrid-agent/blob/v1.3.0/docs/operations.md)
for the complete operational boundary.

## Rollback

Drain and stop the worker, reinstall `v1.2.0`, and restart it with a valid
device-login configuration. Do not replace or mutate existing tags or release
assets; publish further fixes under a new semantic version.
