# rustgrid-agent v1.2.0

This feature release adds the worker-side telemetry model and delivery
foundation for detailed execution and token-consumption reporting.

## Install or upgrade

On macOS with Homebrew:

```sh
brew update
brew upgrade rustgrid-agent
```

For a new installation:

```sh
brew install RustGrid/tap/rustgrid-agent
```

## Telemetry foundation

- Model executions, phases, turns, model calls, and tool calls as versioned
  snapshot events with deterministic identifiers.
- Normalize Codex usage fields, including input, cached-input, output,
  reasoning, and total tokens when the provider reports them.
- Deliver bounded batches to RustGrid's run telemetry endpoint with stable
  idempotency keys.
- Persist pending telemetry in a size-limited outbox so delivery failures stay
  best-effort and do not fail the agent run.
- Continue producing the legacy aggregate token-consumption report during the
  telemetry transition.

The release also refreshes the checked-in RustGrid OpenAPI contract used by the
worker.

## Deployment status

Artifact publication is not production deployment approval. Complete staging
certification against the exact `v1.2.0` image digest before deploying it.

See [known limitations](https://github.com/RustGrid/rustgrid-agent/blob/v1.2.0/docs/known-limitations.md)
and [production operations](https://github.com/RustGrid/rustgrid-agent/blob/v1.2.0/docs/operations.md)
for the complete operational boundary.

## Rollback

Stop the worker and reinstall `v1.1.0`, then restart it with the existing
configuration. Do not replace or mutate existing tags or release assets;
publish further fixes under a new semantic version.
