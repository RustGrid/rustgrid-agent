# rustgrid-agent v1.0.1

This patch release completes container publication after the `v1.0.0` source,
native binary, and Homebrew release succeeded but its container vulnerability
gate correctly stopped the image before it reached GitHub Container Registry.

## Install

On macOS with Homebrew:

```sh
brew update
brew upgrade rustgrid-agent
```

For a new installation:

```sh
brew install RustGrid/tap/rustgrid-agent
```

## Security changes

- The runtime now uses a digest-pinned official Node 24.18 image based on Debian
  trixie instead of Debian bookworm.
- Runtime operating-system packages are upgraded during the image build.
- npm is pinned to 12.0.1, which includes patched `undici` 6.27.0.
- The scanner action runs on Node 24 and keeps Grype pinned to 0.100.0.
- Release logs retain the full vulnerability inventory, while the publication
  gate rejects every High or Critical finding for which a fix is available.

## Deployment status

Artifact publication is not production deployment approval. Complete staging
certification against the exact `v1.0.1` image digest before deploying it.

See [known limitations](https://github.com/RustGrid/rustgrid-agent/blob/v1.0.1/docs/known-limitations.md)
and [production operations](https://github.com/RustGrid/rustgrid-agent/blob/v1.0.1/docs/operations.md)
for the complete operational boundary.

## Rollback

Stop the worker and restore the previously certified internal image or
source-built commit. The `v1.0.0` public container was never published, so it is
not a valid container rollback target. Do not replace or mutate existing tags or
release assets; publish any further remediation under a new semantic version.
